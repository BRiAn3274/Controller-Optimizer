# 兼容性审计

审计目标：让 Controller Optimizer 只修正玩家输入，不替玩家攻击，也不拥有攻击实体的生命周期。

## 已收束项目

| 区域 | 原风险 | 当前边界 |
|---|---|---|
| 激光兼容 | 读取玩家攻击实体并修改 `Timeout`、`Shrink`，会与激光类 Mod 竞争 | 相关路径已全部删除；只输出释放输入 |
| 跟班检测 | 任意 familiar 都会改变玩家本体的 Brimstone 释放 | 只识别原版 Lil Brimstone 和 Lil Monstro；自定义跟班默认放行 |
| 普通 Brimstone | 同时覆盖方向和按下状态会延迟松手 | 只覆盖 `GET_ACTION_VALUE`；`pressed/triggered` 返回 `nil` |
| 里阿撒泻勒 | 需要补足一次触发，但可能重复咳血 | 每次稳定摇杆手势只生成一帧输入触发，持续转向只更新 `pressed/value` |
| 多角色 | 以 `controllerIndex` 共享状态会让同手柄角色互相重置配置 | 使用稳定引擎实体身份按 player 隔离，不能直接依赖临时 userdata wrapper |
| Mars | 覆盖持续 `pressed` 可能影响移动类 Mod | 只抑制确认过的模拟摇杆 `triggered`，`pressed` 始终放行 |
| Schoolbag | 双主动槽即执行，可能误操作特殊角色或 Mod 槽位 | 默认开启；要求真实 Schoolbag、两个主动槽、唯一候选角色，并允许 MCM 关闭 |
| 可选 API | 缺少物理按钮、移动动作或更新回调会禁用全部功能 | 只有射击输入 API 属于核心依赖，其余功能独立失效 |
| 原始输入 | 异常值或读取错误可能让状态机崩溃或刷日志 | 非数字/NaN 归零，数值限制在 0–1，错误日志限频 |

## 仍然存在的兼容性限制

### `MC_INPUT_ACTION` 返回值竞争

这是所有输入 Mod 共用的回调。Controller Optimizer 无法阻止另一个 Mod 对同一玩家、同一 `InputHook`、同一 `ButtonAction` 返回不同值，也无法可靠控制 Workshop 加载顺序。

缓解方式：

- 不相关动作在最前面返回 `nil`；
- 非目标玩家、武器、道具和平台返回 `nil`；
- 普通 Brimstone 不覆盖 `pressed/triggered`；
- 里阿撒泻勒只在 Repentance+ 专用路径中覆盖必要的三个 Hook；
- 所有输出限定为文档允许的布尔值或 0–1 数值。

### 原始输入重采样

状态机通过 `Input.GetActionValue` 读取四个射击轴。该调用可能再次进入输入回调，因此使用递归保护让本 Mod 在内部采样时立即返回 `nil`。其他输入 Mod 仍可能影响采样结果；这是 API 级限制，也是联机或大型输入整合 Mod 需要重点实测的区域。

### 物理按钮编号

Schoolbag 使用 Isaac 物理按钮编号 13（右摇杆按下）。编号 10 是左摇杆按下，11 是右肩键；它们不得触发切换。不同控制器映射或 Steam Input 配置可能改变实际按键；功能默认开启，但可以通过 MCM 关闭。真实 Schoolbag、两个非空主动槽和唯一候选角色是运行前置条件，且该功能不是核心射击修复的依赖。

### 游戏版本变化

Repentance+ 仍可能调整角色输入协议、枚举或回调行为。核心 API 缺失时 Mod 会停止注册输入覆盖并显示失效提示；可选 API 缺失时只关闭对应附加功能。

## 发布前检查

1. `luac -p main.lua`。
2. `lua tests/test_controller_optimizer.lua`。
3. 搜索并确认运行时没有 `FireBrimstone`、`FireDelayedBrimstone`、`Isaac.Spawn`、`GetActiveWeaponEntity` 或 `SetTimeout`。
4. 实测右摇杆回中、单帧掉采样、无中心 snapback、持续 180° 转向和 Analog Stick 全角度旋转。
5. 实测里阿撒泻勒每次手势恰好一次咳血、转向不重复触发、稳定回中后才重新武装。
6. 与至少一个输入覆盖 Mod、一个 Brimstone 修改 Mod 和本地多人场景共同运行，检查日志和攻击方向。
7. 确认 `main.lua` 与 `metadata.xml` 版本一致，再执行 Workshop dry run。
