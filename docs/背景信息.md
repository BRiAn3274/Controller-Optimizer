# Controller Optimizer 背景信息

## 项目定位

Controller Optimizer 是《The Binding of Isaac: Repentance+》的手柄输入修复 Mod，当前稳定版本为 1.7.0，Steam Workshop 项目 ID 为 `3731198516`。

项目最初针对 Steam Deck 和普通手柄的右摇杆问题：摇杆回中、瞬时采样丢失或机械回弹可能让 Brimstone 提前结束、朝错误方向释放，或让里阿撒泻勒的咳血与蓄力输入顺序异常。

## 核心目标

本 Mod 的最终输出是一个更准确的原生输入序列，让游戏自己完成正确的攻击。运行时不得：

- 生成咳血、激光、眼泪、伤害或碰撞实体；
- 调用 `FireBrimstone` 等替玩家攻击的方法；
- 修改激光的持续时间、收缩状态、伤害或充能；
- 在无法确认玩家、角色、武器或输入动作时猜测玩家意图。

允许的介入只有：在 `MC_INPUT_ACTION` 正在读取目标输入时，返回经过死区、回弹和手势状态机处理后的布尔值或 0–1 数值。其他情况返回 `nil`。

## 功能范围

### 核心功能

- 普通 Brimstone：只稳定四方向或 Analog Stick 360° 方向值；`pressed` 和 `triggered` 保持原生。
- 里阿撒泻勒：仅在 Repentance+ 重建一次 `triggered` 启动沿，随后维持实时 `pressed/value`，稳定回中后释放并重新武装。
- 输入滤波：进入/退出死区滞回、两帧采样掉帧保护、突发近 180° 回弹确认。

### 限定兼容功能

- Lil Brimstone / Lil Monstro：只在摇杆回中时提前输出释放，不再读取或修改玩家激光实体。
- Mars：仅在玩家持有 Mars 且确认是模拟摇杆轨迹时抑制触发边沿；持续 `pressed` 保持原生。
- Schoolbag：可选且默认开启，只在玩家确实持有 Schoolbag、拥有两个主动道具、该手柄只对应一个候选角色时调用一次原版交换方法；可在 MCM 中关闭。

## 明确放行

- 非玩家实体和无关 `ButtonAction`；
- `ControllerIndex <= 0` 的键盘输入；
- 非 Brimstone 武器；
- Eye of the Occult；
- 普通 Repentance 下里阿撒泻勒的原生触发；
- 未识别的原版或自定义跟班；
- 缺少 Mars 或 Schoolbag 时的移动与物品输入。

## 维护与发布

- 运行时文件：`main.lua`、`metadata.xml`、`ControllerOptimizer_cover.png`。
- 自动测试：`lua tests/test_controller_optimizer.lua`。
- 语法检查：`luac -p main.lua`。
- 兼容性审计：`docs/兼容性审计.md`。
- 版本改动必须同步 `main.lua`、`metadata.xml`、`CHANGELOG.md` 和发布说明；完成 Repentance+ 实机验证后再上传 Workshop。

重点实机场景包括普通/里阿撒泻勒、Analog Stick、Lil Brimstone、Lil Monstro、Mars、Schoolbag、本地多人、共享控制器角色，以及同时启用其他输入或 Brimstone Mod。
